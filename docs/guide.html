Application guide

From card search to finished deck.

This walkthrough covers the complete deckbuilder workflow: creating or loading a deck, finding cards, organizing both deck zones, saving, and exporting.

01 · Start

Choose what you want to do.

The main menu is the starting point for deck editing and file conversion.

Make a New Deck

Create an empty named deck and open it in the editor.

Load a Deck

Open an editable JSON deck or a human-readable text decklist.

Generate Deck Image from Decklist

Create a Tabletop Simulator sheet directly from a text decklist.

Generate Decklist File

Convert an editable JSON deck into the tournament-friendly text format.

First launch: The application downloads the card database, artwork, and thumbnails. Later launches reuse the local cache.

02 · Search

Find the exact cards you need.

Use any combination of the Card Search controls. The result list updates as filters change, and the card preview updates when you hover, click, or tap a result.

  • Name searches card names as you type.
  • Elements allows one or more element selections.
  • Cost / Level narrows cards by their printed value.
  • Type and Trait use values extracted from the database.
  • Keyword uses declared labels found in current ability text.
  • Expansion limits results to a specific set.
  • Include playtesting cards reveals cards hidden from normal searches.

Choose Clear to reset every filter at once.

03 · Build

Add cards to the main deck or sideboard.

Mouse shortcut

Right-click

Right-click a search result to add one copy to the Main Deck. Hold Shift while right-clicking to add it to the Side Deck.

Drag and drop

Place it directly

Drag a search result into either deck area. The insertion marker shows where the new copy will appear.

The deck model enforces the total zone sizes and the maximum of four copies across the main deck and sideboard. If a card cannot legally be added, the deck remains unchanged.

04 · Organize

Reorder, move, remove, or sort.

  1. Select a batch

    Hold Control on Windows or Linux, or Command on macOS, and click individual card copies in the Main Deck or Side Deck. Selected copies receive a colored border. Starting a selection in the other deck area replaces the current selection.

  2. Move the batch

    Release Control or Command, then left-drag any selected card. Drop the batch at a new position in the same deck area or move it between the Main Deck and Side Deck. Selected cards keep their relative order.

  3. Move one card

    Left-drag an unselected card to reorder it or move it between deck areas.

  4. Remove

    Right-click a deck card, or drag it onto the Card Search panel, to remove one copy.

  5. Sort

    Choose Sort Deck to replace custom ordering with the standard automatic order.

Custom order is intentional: Saving preserves the current visible order. Sorting is only performed when you choose the Sort Deck button.

05 · Save and load

Keep an editable deck or share it.

Save
Updates the current editable JSON file. If the deck has not been saved yet, a save dialog opens.
Save As
Writes the deck to a new JSON file and makes that the active save location.
Open
Loads either an editable JSON deck or a text decklist into the editor.
Rename
Changes the display name and the default filename offered for future exports.

Deck files stay wherever you choose in the system file dialog. They are separate from the application's downloaded card-data cache.

06 · Export

Prepare decks for tournaments or Tabletop Simulator.

Export Decklist

Writes the readable Speedrobo Games format with deck metadata, quantities, sets, totals, and sideboard contents.

Export Main

Creates a Tabletop Simulator PNG sheet for the main deck, including the bundled card back.

Export Sideboard

Creates a matching Tabletop Simulator PNG sheet containing the sideboard.

Install to TTS

Creates the face sheets, installs the bundled MTD card back, and writes a complete saved object into the detected Tabletop Simulator data directory.

One-click installation: Standard Windows, macOS, and Linux TTS locations are detected automatically. If TTS uses a custom data directory, select it once and Caster's Compendium remembers it for later exports. Local assets are intended for single-player or hotseat use; remote players need assets hosted through Steam Cloud or another shared service.

For file conversion without entering the editor, return to the main menu and use the deck-image or decklist generators.

07 · Data and appearance

Keep the card library current.

At startup, the application compares the installed card-list hash with the published hash and can prompt when an update is available. Choose Update Card Database from the main menu whenever you want to force a complete refresh.

Settings lets the application follow the operating-system theme or use a persistent Light or Dark appearance.

The stable application ID remains io.github.hybriduofa.casterdeckbuilder, so upgrading or moving to the new repository does not discard existing cached data.
