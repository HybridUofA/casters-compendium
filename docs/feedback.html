Before submitting

A quick check helps everyone.

  1. Confirm the behavior is still present in the latest release.
  2. Check the application guide for the intended control or workflow.
  3. Search existing issues for the same report or request.
  4. Use a title that identifies the behavior or proposed outcome.
How submission works: The button opens a GitHub Issue Form. A GitHub account is required, and submitting the completed form creates a public issue in the project repository. The Pages site never receives a repository token or stores the form response.
